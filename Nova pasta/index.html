Capítulo 01

HTML: A Estrutura das Coisas

HTML é o esqueleto de toda página web. Antes do estilo, antes do comportamento — existe a estrutura. Entendê-la profundamente é a base de tudo.

🏗
🧠 Analogia Feynman

Pense em uma casa. Antes de pintar as paredes ou instalar a iluminação, você precisa do projeto arquitetônico e das paredes em pé. O HTML é exatamente isso — a planta e a estrutura física da sua interface. O CSS vai pintar. O JavaScript vai instalar os sistemas elétricos.

O que é HTML, de verdade?

HTML significa HyperText Markup Language. Não é uma linguagem de programação — não tem lógica, não faz cálculos. É uma linguagem de marcação: você "marca" pedaços de texto dizendo o que eles são. Um título é um título. Um parágrafo é um parágrafo. Um botão é um botão.

O navegador lê essa marcação e cria o que é chamado de DOM (Document Object Model) — uma árvore de objetos em memória que representa sua página. Tudo que você vai manipular com CSS e JS é, em última análise, essa árvore.

Tags Fundamentais — O que cada uma é e faz

Cada tag tem um propósito semântico — ou seja, não é só sobre aparência, mas sobre significado. Usar as tags certas ajuda na acessibilidade, no SEO e na manutenção do código.

<!DOCTYPE html>

Não é uma tag, é uma declaração. Diz ao navegador: "isso aqui é HTML5, processe com as regras modernas." Sempre na primeira linha.

<html lang="pt-BR">

A raiz. Envolve tudo. O atributo lang diz o idioma — essencial para leitores de tela e SEO.

<head>

Cabeça invisível. Contém metadados, links de CSS, título da aba, fontes. O usuário não vê, mas o navegador usa tudo isso.

<meta charset="UTF-8">

Define a codificação de caracteres. UTF-8 suporta acentos, emojis e praticamente qualquer símbolo. Sem isso, "ação" vira "a??o".

<meta name="viewport">

Crucial para responsividade. Diz ao navegador mobile para não fazer zoom automático. width=device-width é o padrão.

<title>

O texto que aparece na aba do navegador e nos resultados de busca. Seja descritivo: "Login — ERP GENNUS" é melhor que "Página".

<link>

Conecta recursos externos: CSS, fontes, ícones. Atributos essenciais: rel="stylesheet" e href="caminho.css".

<body>

O corpo visível da página. Tudo que o usuário enxerga fica aqui. Filho direto de <html>.

<header>

Cabeçalho da página ou de uma seção. Semanticamente representa a identidade de entrada: logo, nav, hero. Pode aparecer mais de uma vez.

<nav>

Navegação principal. Envolve menus e links de rota. Leitores de tela identificam isso como área de navegação — não use div aqui.

<main>

Conteúdo principal da página. Deve existir apenas uma vez por página. Tudo que é único e relevante entra aqui.

<section>

Uma seção temática do conteúdo. Use quando o bloco tem sentido próprio — funcionalidades, depoimentos, preços. Geralmente tem um heading (h2).

<div>

Container genérico sem significado semântico. Use para agrupamento visual/estilístico quando nenhuma tag semântica fizer sentido. É o canivete suíço do HTML.

<h1> → <h6>

Hierarquia de títulos. h1 é o mais importante (um por página), h6 o menos. Define a estrutura de leitura e SEO.

<p>

Parágrafo. Texto corrido. O navegador adiciona margem automática entre parágrafos — você sobrescreve no CSS.

<span>

Container inline genérico. Use para estilizar parte de um texto sem quebrar o fluxo. Ex: uma palavra com cor diferente.

<a href="">

Link. href é o destino. Use target="_blank" para abrir em nova aba (e adicione rel="noopener" por segurança).

<img src="" alt="">

Imagem. O alt é obrigatório para acessibilidade — descreve a imagem para quem não pode vê-la. Tag auto-fechante.

<form>

Formulário. Agrupa inputs e o botão de envio. Atributos: action (onde enviar), method (GET ou POST). No frontend moderno, o envio é feito via JS.

<input type="">

Campo de entrada. O type muda tudo: text, email, password, checkbox, file, number. Tag auto-fechante.

<label>

Rótulo de um input. Conecte com for="id-do-input". Clicar no label já foca o campo — melhora muito a usabilidade.

<button type="">

type="submit" envia o form. type="button" não faz nada por padrão — ideal para ações controladas por JS. Sempre use type.

<footer>

Rodapé. Contém créditos, links legais, redes sociais. Pode ser global ou de uma seção.

<script src="">

Carrega ou executa JavaScript. Coloque sempre no final do <body> — assim o HTML carrega antes do script tentar manipulá-lo.

Esqueleto de uma Landing Page

Abaixo o esqueleto comentado de uma landing page profissional — o padrão que você vai usar em praticamente qualquer projeto:

HTML · landing-page.html 
<!-- Declaração obrigatória. Sempre primeira linha. -->
<!DOCTYPE html>
<html lang="pt-BR">
 
<head>
  <!-- Codificação: suporte a acentos e caracteres especiais -->
  <meta charset="UTF-8">
 
  <!-- Viewport: essencial para responsividade mobile -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
 
  <!-- Título que aparece na aba do navegador e no Google -->
  <title>ERP GENNUS — Gestão Inteligente</title>
 
  <!-- Conecta o arquivo de estilos externo -->
  <link rel="stylesheet" href="styles.css">
 
  <!-- Fonte do Google Fonts -->
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700&display=swap"
        rel="stylesheet">
</head>
 
<body>
 
  <!-- CABEÇALHO: logo + navegação -->
  <header class="site-header">
    <div class="container">
      <a href="#" class="logo">GENNUS</a>
      <nav class="main-nav">
        <a href="#funcionalidades">Funcionalidades</a>
        <a href="#planos">Planos</a>
        <a href="login.html" class="btn btn-outline">Login</a>
        <a href="cadastro.html" class="btn btn-primary">Começar Grátis</a>
      </nav>
      <!-- Botão hambúrguer para mobile (controlado por JS) -->
      <button class="menu-toggle" type="button" aria-label="Abrir menu">
        <span></span><span></span><span></span>
      </button>
    </div>
  </header>
 
  <!-- CONTEÚDO PRINCIPAL -->
  <main>
 
    <!-- HERO: primeira seção, o "gancho" da página -->
    <section class="hero">
      <div class="container">
        <h1>Gestão que acompanha o ritmo do seu negócio</h1>
        <p class="hero-sub">ERP inteligente, em tempo real, para quem não tem tempo a perder.</p>
        <a href="cadastro.html" class="btn btn-primary btn-lg">Começar Gratuitamente</a>
      </div>
    </section>
 
    <!-- FUNCIONALIDADES: cards com features do sistema -->
    <section id="funcionalidades" class="features">
      <div class="container">
        <h2>Tudo que você precisa em um só lugar</h2>
        <div class="features-grid">
          <!-- Card de funcionalidade (repete N vezes) -->
          <div class="feature-card">
            <img src="icons/dashboard.svg" alt="Dashboard">
            <h3>Dashboard em Tempo Real</h3>
            <p>Visualize métricas críticas do seu negócio em segundos.</p>
          </div>
        </div>
      </div>
    </section>
 
  </main>
 
  <!-- RODAPÉ -->
  <footer class="site-footer">
    <div class="container">
      <p>&copy; 2025 ERP GENNUS. Todos os direitos reservados.</p>
    </div>
  </footer>
 
  <!-- Scripts no final do body: HTML já carregou antes de executar -->
  <script src="main.js"></script>
 
</body>
</html>

Esqueleto de uma Página de Login

<body class="auth-page">
 
  <div class="auth-container">
 
    <!-- Lado esquerdo: branding/hero (visível em desktop) -->
    <div class="auth-brand">
      <a href="index.html" class="logo">GENNUS</a>

    <!-- Lado direito: formulário de login -->
    <div class="auth-form-panel">
      <div class="form-card">
        <h1>Entrar</h1>
 
        <!-- novalidate: desabilita validação nativa, usamos a nossa via JS -->
        <form id="login-form" novalidate>
 
          <div class="form-group">

            <input
              type="email"
              id="email"
              name="email"
              placeholder="seu@email.com"
              autocomplete="email"
              required
            >
            <!-- Mensagem de erro para este campo (controlada por JS) -->
            <span class="field-error" id="email-error"></span>
          </div>
 
          <div class="form-group">
            <label for="senha">Senha</label>
            <div class="input-wrapper">
              <input
                type="password"
                id="senha"
                name="senha"
                placeholder="••••••••"
                autocomplete="current-password"
                required
              >
              <!-- Ícone de olho para mostrar/ocultar senha (controlado por JS) -->
              <button type="button" class="toggle-password" aria-label="Mostrar senha">
                👁
              </button>
            </div>
            <span class="field-error" id="senha-error"></span>
          </div>
 
          <div class="form-actions">
            <a href="#" class="link-subtle">Esqueci minha senha</a>
            <button type="submit" class="btn btn-primary btn-full">
              Entrar
            </button>
          </div>
 
        </form>
 
        <p class="auth-redirect">
          Não tem conta? <a href="cadastro.html">Criar conta grátis</a>
        </p>
 
      </div>
    </div>
 
  </div>
 
  <script src="login.js"></script>
</body>
Prefira IDs únicos (id="") para campos de formulário — o label for="" precisa deles. Use classes (class="") para estilização com CSS que se repete em vários elementos.
Capítulo 02

CSS: O Estilo e a Alma Visual

CSS transforma o esqueleto HTML em uma interface que as pessoas querem usar. É onde cores, espaços, tipografia e animações se tornam experiência.

🎨
🧠 Analogia Feynman

Se HTML é a estrutura da casa, CSS é o design de interiores: cor das paredes, tipo de piso, iluminação, onde cada móvel fica. A mesma planta (HTML) pode virar um apartamento minimalista ou um quarto de hotel cinco estrelas — tudo depende do CSS.

Reset e Base — O Ponto de Partida

Todo navegador tem estilos padrão embutidos — margens, paddings, fontes. O primeiro passo de qualquer projeto é limpar esses defaults e definir sua própria base:

CSS · base.css 
/* ===== RESET UNIVERSAL ===== */
*,
*::before,
*::after {
  margin: 0;
  padding: 0;
  box-sizing: border-box;   /* padding/border contam DENTRO da largura */
}
 
/* ===== VARIÁVEIS GLOBAIS (Custom Properties) ===== */
:root {
  /* Cores */
  --color-bg: #0d0d1a;
  --color-surface: #1e1e3f;
  --color-border: #2a2a5a;
  --color-accent: #6c63ff;
  --color-text: #e2e8f0;
  --color-text-muted: #94a3b8;
 
  /* Tipografia */
  --font-sans: 'Inter', system-ui, sans-serif;
  --font-mono: 'JetBrains Mono', monospace;
 
  /* Espaçamentos base */
  --space-xs: 4px;
  --space-sm: 8px;
  --space-md: 16px;
  --space-lg: 32px;
  --space-xl: 64px;
 
  /* Bordas */
  --radius: 12px;
  --radius-sm: 8px;
  --radius-full: 9999px;
 
  /* Transições */
  --transition: all 0.25s ease;
}
 
/* ===== BASE GLOBAL ===== */
html {
  scroll-behavior: smooth;   /* scroll suave ao clicar em âncoras */
  font-size: 16px;            /* base para cálculos rem */
}
 
body {
  font-family: var(--font-sans);
  background-color: var(--color-bg);
  color: var(--color-text);
  line-height: 1.6;
  -webkit-font-smoothing: antialiased; /* textos mais suaves no Mac */
}
 
img {
  max-width: 100%;   /* imagens nunca ultrapassam o container */
  display: block;
}
 
/* ===== CONTAINER PADRÃO ===== */
.container {
  width: 100%;
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 24px;
}

Responsividade — Media Queries

CSS · responsive.css 
/* ===== BREAKPOINTS PADRÃO ===== */
/* Mobile primeiro (mobile-first): estilos base = mobile,
   media queries adicionam comportamento em telas maiores */
 
/* Tablet: 768px+ */
@media (min-width: 768px) {
  .features-grid {
    grid-template-columns: repeat(2, 1fr);
  }
  .menu-toggle { display: none; }
  .main-nav    { display: flex; }
}
 
/* Desktop: 1024px+ */
@media (min-width: 1024px) {
  .features-grid {
    grid-template-columns: repeat(3, 1fr);
  }
  .auth-container {
    display: grid;
    grid-template-columns: 1fr 1fr;
  }
}
 
/* Mobile (base, sem media query necessária): */
.main-nav { display: none; } /* oculto no mobile */
.menu-toggle { display: flex; } /* hambúrguer aparece no mobile */
 
/* ===== UNIDADES: REM vs PX ===== */
/*
  px  → absoluto, não escala com preferências do usuário
  rem → relativo ao html (16px base), escala bem, acessível
  em  → relativo ao elemento pai, útil para componentes isolados
  %   → relativo ao container pai
  vw/vh → relativo à viewport (janela)
 
  Regra prática:
  - font-size → use rem
  - padding/margin → use rem ou px
  - larguras de layout → use %, fr, ou px com max-width
  - alturas de seções → use vh ou px
*/
Capítulo 03

JavaScript: O Comportamento e a Vida da Página

HTML estrutura, CSS estiliza. JavaScript faz acontecer. É a única das três linguagens que realmente programa — tem lógica, memória e reage ao usuário.

⚡
🧠 Analogia Feynman

Na sua casa: HTML é a estrutura, CSS é o design. O JavaScript são os sistemas elétrico, hidráulico e de automação — tudo que responde quando você aperta um botão, liga uma luz ou abre uma torneira. Sem ele, a casa é bonita mas estática.

Variáveis: let, const e var

JS · variaveis.js 
// ===== CONST: valor que não muda =====

Validação de Formulário Completa

JS · validacao.js — Página de Cadastro 
// ===== VALIDAÇÃO DE FORMULÁRIO DE CADASTRO ===== 
 
// Selecionamos todos os campos que vamos usar
const form          = document.querySelector('#cadastro-form');
const nomeInput     = document.querySelector('#nome');
const emailInput    = document.querySelector('#email');
const senhaInput    = document.querySelector('#senha');
const confirmInput  = document.querySelector('#confirmar-senha');
 
// ─── Funções auxiliares ────────────────────────────────────
 
// Exibe ou limpa o erro de um campo
const setError = (input, mensagem) => {
  const errorEl = document.querySelector(`#${input.id}-error`);
  if (mensagem) {
    input.classList.add('error');
    input.classList.remove('valid');
    if (errorEl) errorEl.textContent = mensagem;
  } else {
    input.classList.remove('error');
    input.classList.add('valid');
    if (errorEl) errorEl.textContent = '';
  }
};
 
// Valida e-mail com regex
const isEmailValido = (email) => {
  const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return regex.test(email);
};
 
// Valida cada campo individualmente
const validarNome = () => {
  const v = nomeInput.value.trim();
  if (!v) return setError(nomeInput, 'Nome é obrigatório');
  if (v.length < 2) return setError(nomeInput, 'Mínimo 2 caracteres');
  setError(nomeInput, null); // limpa o erro
  return true;
};
 
const validarEmail = () => {
  const v = emailInput.value.trim();
  if (!v) return setError(emailInput, 'E-mail é obrigatório');
  if (!isEmailValido(v)) return setError(emailInput, 'E-mail inválido');
  setError(emailInput, null);
  return true;
};
 
const validarSenha = () => {
  const v = senhaInput.value;
  if (!v) return setError(senhaInput, 'Senha é obrigatória');
  if (v.length < 8) return setError(senhaInput, 'Mínimo 8 caracteres');
  setError(senhaInput, null);
  return true;
};
 
const validarConfirmacao = () => {
  if (confirmInput.value !== senhaInput.value) {
    return setError(confirmInput, 'As senhas não coincidem');
  }
  setError(confirmInput, null);
  return true;
};
 
// ─── Validação ao sair do campo (blur) ────────────────────
nomeInput.addEventListener('blur', validarNome);
emailInput.addEventListener('blur', validarEmail);
senhaInput.addEventListener('blur', validarSenha);
confirmInput.addEventListener('blur', validarConfirmacao);
 
// ─── Validação final no submit ─────────────────────────────
form.addEventListener('submit', (e) => {
  e.preventDefault();
 
  const okNome     = validarNome();
  const okEmail    = validarEmail();
  const okSenha    = validarSenha();
  const okConfirm  = validarConfirmacao();
 
  if (okNome && okEmail && okSenha && okConfirm) {
    console.log('✅ Formulário válido! Enviar para a API.');
    // aqui vai a chamada fetch() para a API
  }
});

// Adiciona efeito de entrada animada ao rolar a página (Intersection Observer)
const cards = document.querySelectorAll('.feature-card');
 
const observer = new IntersectionObserver((entries) => {
  entries.forEach((entry, i) => {
    if (entry.isIntersecting) {
      // Delay escalonado: cada card aparece 100ms depois do anterior
      setTimeout(() => {
        entry.target.classList.add('visible');
      }, i * 100);
      observer.unobserve(entry.target); // para de observar depois de animar
    }
  });
}, { threshold: 0.1 }); // dispara quando 10% do card está visível

Princípios de UI que Você Precisa Internalizar

PrincípioO que éComo aplicar
Hierarquia VisualO olho segue uma ordem. O mais importante precisa ser o mais destacado.Tamanho de fonte: h1 grande, h2 médio, p pequeno. Contraste: cor forte em CTAs, sutil em texto secundário.
EspaçamentoEspaço em branco não é desperdício — é respiração. Elementos agrupados próximos comunicam relação.Use um sistema: gap de 8px entre elementos internos, 32px entre seções. Nunca espaços aleatórios.
ContrasteTexto ilegível é design ruim. Ratio mínimo recomendado: 4.5:1 (WCAG AA).Texto branco puro (#fff) no fundo escuro (#0d0d1a) tem contraste excelente. Evite cinza claro em cinza escuro.
ConsistênciaBotões primários têm sempre o mesmo estilo. Títulos têm sempre a mesma fonte. O usuário aprende seu sistema.Use CSS variables. Crie classes reutilizáveis (.btn, .card, .badge). Nunca estilos inline repetidos.
FeedbackO usuário precisa saber que aconteceu algo quando clica ou preenche.Hover com transição, estado de loading no botão, mensagem de erro clara, estado de sucesso visível.

🚀 Próximos Passos: Para Além do Vanilla

Você vai chegar em um ponto onde projetos maiores vão pedir mais organização. Aqui está o mapa:

  • Git e GitHub — Controle de versão. Essencial para qualquer projeto profissional. Comece aqui antes de frameworks.
  • Node.js + NPM — Ecossistema de ferramentas. Vai permitir usar bundlers, linters e a maioria das libs modernas.
  • React ou Vue — Frameworks JavaScript para interfaces complexas e reativas. React domina o mercado de emprego. Vue tem curva de aprendizado menor. Aprenda depois de dominar o vanilla.
  • Tailwind CSS — Framework utilitário de CSS. Extremamente popular em projetos React/Vue. Aprenda depois de entender bem o CSS nativo (que você já está fazendo).
  • TypeScript — JavaScript com tipos estáticos. Padrão em empresas maiores. Aprenda JavaScript puro primeiro.
  • Vite ou Next.js — Ferramentas de build modernas. Vite para projetos simples, Next.js para projetos React com SSR.

A base que você construiu aqui — HTML semântico, CSS em controle, JS funcional — é o que separa devs que entendem de devs que apenas copiam e colam de frameworks. Frameworks mudam. Os fundamentos ficam.
